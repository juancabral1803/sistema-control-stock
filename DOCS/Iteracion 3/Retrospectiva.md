## Iteración 3 – Información general

**Proyecto:** Sistema de Control de Stock  
**Iteración:** 3  
**Duración:** 3 semanas  
**Objetivo:** Optimizar el rendimiento del sistema, implementar control de acceso y mejorar la interfaz de usuario.

---

### Aspectos positivos

- Se cumplieron los objetivos planificados: mejoras de rendimiento, seguridad y usabilidad.  
- Las consultas a la base de datos fueron optimizadas con éxito, logrando una reducción del tiempo de carga del 40 %.  
- Se integró correctamente el módulo de **autenticación y control de roles** con Spring Security.  
- La interfaz fue rediseñada con un estilo más moderno, responsivo y coherente con las funciones del sistema.  
- La colaboración entre las parejas **XP** fue fluida, y el intercambio de roles fomentó el aprendizaje cruzado entre integrantes.  
- Se fortaleció la organización del repositorio, la documentación técnica y el uso disciplinado de Git.

---

### Aspectos a mejorar

- Durante las pruebas de integración, se detectaron conflictos de dependencias entre librerías externas (Spring Security y PDF export).  
- El tiempo destinado a diseño visual fue limitado; se podrían aplicar mejoras de **accesibilidad y experiencia de usuario**.  
- Algunas pruebas automatizadas quedaron pendientes por falta de tiempo.  
- Hubo pequeños retrasos en los commits finales por diferencias de formato entre ramas.

---

### Acciones de mejora

- Planificar una iteración de mantenimiento post-entrega para incorporar pruebas adicionales y mejoras visuales.  
- Documentar el flujo de **integración continua (CI/CD)** para futuros proyectos.  
- Implementar un pipeline de testing automatizado (**JUnit + GitHub Actions**).  
- Evaluar la adopción de un framework de frontend moderno (por ejemplo, React o Vue) para una futura versión.

---

### Lecciones aprendidas

- La aplicación de **Scrum y XP** en conjunto permitió organizar el trabajo de forma ágil y colaborativa, manteniendo entregas frecuentes y medibles.  
- Las reuniones de retrospectiva fueron clave para detectar errores a tiempo y mejorar procesos en cada iteración.  
- El enfoque iterativo demostró que es más eficiente desarrollar un sistema de manera incremental que intentar hacerlo completo desde el inicio.  
- La práctica del **pair programming (driver/navigator)** mejoró la calidad del código y redujo los errores.  
- La documentación continua en GitHub facilitó la trazabilidad y comunicación entre los miembros del equipo.

---

### Conclusión general del proyecto

El proyecto **Sistema de Control de Stock** alcanzó todos los objetivos propuestos a lo largo de las tres iteraciones.  
El producto final es funcional, seguro, optimizado y fácil de usar, cumpliendo con los requerimientos del cliente y con los principios de las metodologías ágiles.  

El equipo demostró capacidad para adaptarse, resolver problemas técnicos y colaborar de forma efectiva.  
A través del uso de **Scrum y Extreme Programming**, se logró entregar un sistema realista, bien documentado y alineado con las buenas prácticas de desarrollo de software.
