## Iteración 1 – Información general

**Proyecto:** Sistema de Control de Stock  
**Iteración:** 1  
**Duración:** 3 semanas  
**Objetivo:** Implementar el registro de productos nuevos (**HU1**) y el registro de salidas de productos (**HU2**)

---

### Aspectos positivos

- El equipo logró cumplir con las funcionalidades planificadas dentro del tiempo estimado.  
- Se trabajó correctamente bajo la metodología **Scrum**, realizando reuniones breves diarias para seguimiento del progreso.  
- Las parejas **XP** funcionaron de forma efectiva, alternando roles sin inconvenientes.  
- Se configuró correctamente el entorno de trabajo (**Spring Boot + MySQL + GitHub**).  
- Las pruebas unitarias con **JUnit** permitieron detectar y corregir errores tempranamente.  
- Buena comunicación entre el **Product Owner** y el equipo de desarrollo para aclarar requisitos y criterios de aceptación.

---

### Aspectos a mejorar

- La planificación inicial subestimó el tiempo requerido para configurar la base de datos y la conexión JDBC.  
- Algunos commits no se realizaron siguiendo la convención acordada, generando confusión al revisar el repositorio.  
- Falta definir una guía de estilo común para el código (formato, comentarios, nombres de métodos).  
- No se llegó a implementar un sistema de **logs** para depuración.

---

### Acciones de mejora

- Agregar una **checklist de configuración** para nuevos entornos de desarrollo.  
- Establecer una **convención de commits** y una política clara de ramas (`main / develop / feature`).  
- Implementar una **plantilla de estilo de código Java** (Google Style o similar).  
- Planificar con mayor precisión el tiempo de tareas técnicas en la próxima iteración.  
- Incluir pruebas automatizadas adicionales para la inserción y eliminación de datos.

---

### Lecciones aprendidas

- La importancia de una buena definición de **criterios de aceptación** antes de comenzar a codificar.  
- El valor de las reuniones diarias breves (**Daily Scrum**) para mantener sincronizado al equipo.  
- La ventaja del enfoque **XP** en la rotación de roles `driver/navigator`, que mejora la comprensión compartida del código.  
- La necesidad de mantener una **documentación actualizada** en el repositorio GitHub durante toda la iteración.

---

### Conclusión

La **Iteración 1** fue exitosa: se cumplieron los objetivos principales y se estableció una base sólida para las próximas entregas.  
El equipo adquirió experiencia en la aplicación práctica de **Scrum y XP**, y definió estrategias concretas para optimizar la organización y calidad del trabajo en la Iteración 2.
